Section 2: Confidence Distribution Analysis¶

Motivation¶

Notebook 4.0 Section 7 reported aggregate F1 but said nothing about confidence. A model can be right for the wrong reasons — high confidence on easy cases, near-random on hard ones. This section asks whether the model's confidence is calibrated: do high-confidence predictions actually have higher accuracy than low-confidence ones?

Analysis¶

  • Compute prediction confidence (max class probability) for all 200 validation cases
  • Bin predictions by confidence range and compute accuracy per bin
  • Compare confidence distributions for correct vs incorrect predictions
  • Flag any cases where the model is wrong with high confidence — these are the most dangerous failure modes in production

Section 3: Emoji Feature Contribution Analysis¶

Motivation¶

Notebook 4.0 Section 8 showed that emoji_neg_count and emoji_pos_count have surprisingly weak coefficients (+0.15 and -0.05) compared to word features. This section asks what that means in practice — how often does emoji signal actually change the prediction, and in which direction?

Analysis¶

  • For each validation case, compute prediction with and without numeric features (emoji + word counts)
  • Measure how many predictions flipped when numeric features were removed
  • Isolate the emoji-only contribution by zeroing word counts
  • Characterize the cases where emoji signal was decisive

# Build three versions of the validation matrix:
# 1. Full features  — text + emoji + word
# 2. Text only      — no numeric features
# 3. Text + emoji   — no word counts (isolate emoji contribution)

def zero_cols(matrix, col_indices):
    """Return a copy of matrix with specified columns zeroed."""
    m = matrix.toarray().copy()
    for i in col_indices:
        m[:, i] = 0
    from scipy.sparse import csr_matrix
    return csr_matrix(m)

n_text = X_vl_text_vec.shape[1]  # 1265

# Column indices in final matrix
# [0..1264] = tfidf, 1265=emoji_pos, 1266=emoji_neg, 1267=word_pos, 1268=word_neg
IDX_EMOJI_POS = n_text + 0
IDX_EMOJI_NEG = n_text + 1
IDX_WORD_POS  = n_text + 2
IDX_WORD_NEG  = n_text + 3

X_text_only      = zero_cols(X_val_final, [IDX_EMOJI_POS, IDX_EMOJI_NEG,
                                            IDX_WORD_POS,  IDX_WORD_NEG])
X_text_emoji     = zero_cols(X_val_final, [IDX_WORD_POS, IDX_WORD_NEG])
X_text_word      = zero_cols(X_val_final, [IDX_EMOJI_POS, IDX_EMOJI_NEG])

y_text_word  = model.predict(X_text_word)

    return accuracy_score(y_val, preds) - acc

Section 4: Misclassification Archetype Analysis¶

Motivation¶

Notebook 4.0 Section 9 identified 35 misclassifications and flagged qualitative patterns — short ambiguous tweets, sarcasm without emoji, positive words with negative intent. This section formalizes those patterns into archetypes with counts and examples.

Archetypes¶

  • Lexical trap — positive words in negative context (e.g. "great network" in a critical tweet)
  • Minimal context — fewer than 4 tokens, no sentiment signal (e.g. "Late nights", "beauty", "but no sign.")
  • Emoji-text conflict — handled by veto at inference, visible here as raw model errors
  • Sarcasm without emoji — ironic phrasing, no symbolic signal to trigger veto

# ================================================================
# Section 4: Misclassification Archetype Analysis
# ================================================================

misclassified = results_df[~results_df["correct"]].copy()

# Archetype classification rules
def classify_archetype(row):
    text  = str(row["text"]).lower()
    tokens = text.split()

    has_emoji_signal = any(
        c in row["text"] for c in "😭😊😍😧😂❤️😁😘😩😤"
    )
    has_positive_word = any(
        w in text for w in ["great", "good", "happy", "love",
                             "amazing", "perfect", "nice", "awesome"]
    )
    is_short = len(tokens) <= 4

    if has_emoji_signal:
        return "emoji-text conflict"
    elif is_short:
        return "minimal context"
    elif has_positive_word and row["true"] == 0:
        return "lexical trap"
    else:
        return "sarcasm / ambiguous"

misclassified["archetype"] = misclassified.apply(classify_archetype, axis=1)

archetype_summary = misclassified.groupby("archetype").agg(
    count=("archetype", "count"),
    avg_confidence=("confidence", "mean"),
).sort_values("count", ascending=False)

print("Misclassification Archetypes:")
print(f"{'Archetype':<25} {'Count':<8} {'Avg Confidence'}")
print("-" * 48)
for name, row in archetype_summary.iterrows():
    print(f"{name:<25} {int(row['count']):<8} {row['avg_confidence']:.4f}")

print(f"\nSample cases by archetype:")
for archetype in archetype_summary.index:

Section 6: Summary and Findings¶

Confidence Calibration¶

The model is well-calibrated where it matters. The 0.50–0.60 confidence bin is the honest failure zone — 59 cases, 55.93% accuracy, barely above chance. These are inputs the model genuinely cannot resolve. The 0.90–1.00 bin holds 92 cases at 97.83% accuracy — high confidence is earned, not assumed.

Two high-confidence errors exist at 0.986 and 0.984. Both are sarcasm cases — "I hate people who steal my ideas" predicted Positive with near certainty. These are the most consequential failure mode in production: wrong with conviction.

Emoji Feature Contribution¶

Word counts drive prediction changes more than emoji counts. Removing all numeric features drops accuracy by 7.5% (25 flips). Removing word counts alone causes 0 accuracy delta. Removing emoji counts alone causes a -0.005 delta and 3 flips.

This confirms the coefficient story from Notebook 4.0: word_neg_count at -4.08 is the dominant numeric feature. Emoji signal matters most through the deterministic veto — not through model coefficients.

Misclassification Archetypes¶

35 misclassifications across four archetypes:

  • Sarcasm / ambiguous (17) — dominant failure mode. Average confidence 0.6188 — the model is uncertain but still wrong. One case at 0.986 confidence is the dangerous outlier.
  • Minimal context (15) — honest failures. Tweets like "Late nights", "beauty", "but no sign." carry no learnable signal. Average confidence 0.5661 — the model knows it doesn't know.
  • Lexical trap (2) — positive words in negative context. Low count suggests the word lexicon is not over-triggering.
  • Emoji-text conflict (1) — expected. The veto handles these at inference time. Raw model errors here are not production errors.

Next Steps¶

Notebook 4.5 is closed. The model is understood.

The dominant failure mode — sarcasm and ambiguous phrasing at 17/35 misclassifications — is not addressable through feature engineering or veto refinement. The two high-confidence errors are the most actionable finding: cases the model is wrong about with conviction are the ones most likely to cause visible production failures.

If this project continues, the next directions are:

More training data — the correct lever for sarcasm and minimal context failures. Both archetype counts would shrink with a larger, more diverse corpus. Feature redesign is not the answer here.

Confidence thresholding — flagging predictions below 0.60 as low-confidence in the API response gives downstream consumers the information they need to handle uncertain cases differently. The calibration analysis shows 0.60 is the honest boundary.

Deployment — predict_sentiment() is stable, verified, and production-ready. The inference layer is fully synchronized with notebook specifications. Application layer integration can proceed.
